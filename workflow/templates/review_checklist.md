# フェーズ別レビューチェックリスト

> 各フェーズのレビュー時に本チェックリストを使用してください。
> 全項目を確認し、チェックを付けてからフェーズ完了としてください。

---

## Phase 1: 要件・判断 レビュー

- [ ] ビジネス要件が網羅的に整理されているか
- [ ] 非機能要件（可用性、性能、セキュリティ）が定量化されているか
- [ ] ScalarDB適用判断のデシジョンツリーに従って判断されているか
- [ ] XA vs ScalarDBの比較が客観的に行われているか
- [ ] 境界コンテキストが適切に分割されているか（大きすぎず小さすぎず）
- [ ] ScalarDB管理対象テーブルが最小化されているか
- [ ] 2PC適用範囲が制限ガイドラインに従っているか（最大2-3サービス）

---

## Phase 2: 設計 レビュー

- [ ] データモデルがScalarDBの制約を考慮しているか（PK/CK/SI設計）
- [ ] Partition Keyのホットスポットリスクが評価されているか
- [ ] メタデータオーバーヘッドが見積もられているか
- [ ] トランザクションパターンが適切に選定されているか
- [ ] OCC競合率が許容範囲か（< 5%推奨（設計段階の目安。テスト戦略 `12_testing_strategy.md` での目標値と統一））
- [ ] バッチ処理のチャンクサイズが適切か
- [ ] API設計がScalarDBトランザクション例外を適切にハンドリングしているか
- [ ] CDCメタデータ（tx_state）のフィルタリングが設計されているか

---

## Phase 3: 基盤 レビュー

- [ ] ScalarDB Clusterのレプリカ数が可用性要件を満たしているか
- [ ] PodDisruptionBudget / Anti-Affinityが設定されているか
- [ ] TLS/mTLSが全通信経路で有効化されているか
- [ ] RBACが最小権限原則に基づいているか
- [ ] Coordinatorテーブルが保護されているか
- [ ] 監査ログの暫定措置が実装されているか
- [ ] メトリクス/ログ/トレースの3本柱が設計されているか
- [ ] アラート閾値が設定されているか
- [ ] RPO/RTOが現実的な値に設定されているか
- [ ] バックアップ・リストア手順がテスト済みか
- [ ] DR手順書（ランブック）が作成されているか

---

## Phase 4: 実行 レビュー

- [ ] 実装タスクに優先順位がつけられているか
- [ ] ScalarDB固有の実装パターンが正しく適用されているか
- [ ] トランザクション内のエラーハンドリング（リトライ、abort）が実装されているか
- [ ] テストピラミッドに基づいたテストが作成されているか
- [ ] 2PC障害シナリオのテストが含まれているか
- [ ] 性能テストの目標値と結果が記録されているか
- [ ] デプロイ手順書にロールバック手順が含まれているか
- [ ] Go/No-Go判断基準が定義されているか
- [ ] スキーママイグレーション手順が後方互換性を考慮しているか

---

## クロスカッティング チェック

- [ ] 全ドキュメントで用語が統一されているか（ユビキタス言語）
- [ ] 「分散モノリス」リスクが回避されているか（過度な2PC適用がないか）
- [ ] ScalarDB管理対象の最小化原則が守られているか
- [ ] DB固有機能の制限が関係者に共有されているか
- [ ] ScalarDB 3.17の最適化機能が活用されているか（Piggyback Begin, Write Buffering, Batch Operations）
- [ ] Transaction Metadata DecouplingのREADレイテンシ影響が評価されているか
